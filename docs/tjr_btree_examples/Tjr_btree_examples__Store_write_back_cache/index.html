Module Tjr_btree_examples__Store_write_back_cache

NOTE alloc is necessary because a write of a node may only allocate the block, rather than actually writing.

NOTE that eviction should be handled asynchronously, by repeatedly calling wrte on the lower store_ops. Obviously evicted writes should not be overtaken by writes to the same blocks (the lower store should be accessed serially).
